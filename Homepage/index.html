Architecture

How It Works

Four-stage pipeline from attack input to defense response, all running locally on your machine.

01
Attack Input

Select a threat vector or craft a custom adversarial prompt in the Attack Lab

02
Defense Gate

FastAPI backend analyzes the prompt through the neural defense matrix when enabled

03
LLM Processing

Payload reaches local Ollama LLaMA model — defended or raw depending on toggle state

04
Breach Report

Response analyzed, breach detected or neutralized, full event logged to console
